# Wave Function Conventions and Spin-Isospin Operators
## Wave Function Spin-Isospin Elements
The wave function is stored as a matrix. Each row represents an isospin configuration and each column a spin
configuration. For example, the wave function matrix for 2H is:

```
2H Wave function:

     dd   du   ud   uu
   -------------------
np | a1   a2   a3   a4
pn | a5   a6   a7   a8

n - neutron
p - proton
u - spin up
d - spin down
a - matrix element
```
2H is composed of two particles a neutron and a proton. The first matrix element, a1, 
is the amplitude of the following spin-isospin configuration:

a1 spin-isospin configuration:
* particle 1 spin down proton
* and particle 2 spin down neutron

Another example:

a7 spin-isospin configuration:
* particle 1 spin down neutron
* and particle 2 spin up proton

Both the spin and isospin configurations of the columns and rows are in increasing order
in binary representation (e.g., udu->101 or ppn->110). The matrix elements can be complex or real valued.


## Spin And Isospin operators <img src="https://render.githubusercontent.com/render/math?math=\sigma_{ij} \quad \tau_{ij}">
### Introduction
Both operators perform the same operation of the wave function matrix described above. The difference is that 
<img src="https://render.githubusercontent.com/render/math?math=\sigma_{ij}"> operates on the spin space (the columns of
the wave function) and <img src="https://render.githubusercontent.com/render/math?math=\tau_{ij}"> operates on the 
isospin space (the rows of the wave function). 

The operation of these two operators on the wave function is described in equation 34 of 
[Quantum Monte Carlo methods for nuclear physics](https://arxiv.org/abs/1412.3081) but in our case the spin and isospin
spaces are separated into columns and rows respectively.

### Implementation
The function of both operators on the wave function have the following form:
<img src="https://render.githubusercontent.com/render/math?math=\hat{O} \psi = 2 \psi' - \psi"> where
psi is the matrix described above. Psi prime is generated by swapping elements using pre-generated indices.
For each pair of particles i and j a set of indices are created. The indices for sigma swap columns while the indices
for tau swap rows of the wave function. These indices are generated once at the beginning of each run. 
In rough numpy pseudo code both operators follow these steps:

1. At code start generate indices for each pair of particles for both tau and sigma
2. substitute indices into psi to get psi prime: 
for sigma: <img src="https://render.githubusercontent.com/render/math?math=\psi'=\psi[:, sigma\_indices]">
for tau: <img src="https://render.githubusercontent.com/render/math?math=\psi'=\psi[tau\_indices]">
3. return <img src="https://render.githubusercontent.com/render/math?math=\sigma \psi'=2*\psi'-\psi">

The indices substitution is generally done all at once. The code as written is capable of doing any number of pair
operations at a given time (e.g. one at a time or all at once) which is controlled by how many indices are added
to the arguments of the sigma and tau functions. Note for developers: changing how many operations are done at one time 
can help you manage time and space complexity.

# Wave Function Antisymmetry
Antisymmetry in the code is enforced by representing the wave function 
as a sum over products of symmetric and antisymmetric terms:

<img src="https://render.githubusercontent.com/render/math?math=\Psi = \sum_i S_i A_{imn}">


The symmetric parts <img src="https://render.githubusercontent.com/render/math?math=S_i"> 
are typically just scalars but the antisymmetric parts <img src="https://render.githubusercontent.com/render/math?math=A_{imn}">
(m for row and n for column)
are matrices (see wave function convention above). The matrix is forced to be antisymmetric by its construction
at the beginning of the code.

### Construction of The Antisymmetric Matrix A
The wave function is fully specified by the user at the beginning of the calculation (see input specification for 
details). Each orbitals string contains the radial, angular, spin, and isospin numbers for each particle. For example
the deuteron in the ground state might have would be given:
~~~json
    "orbitals": [
      [
        "R0_Y00_d_n",
        "R0_Y00_d_p"
      ]
    ],
~~~
R is the radial function, Y the real spherical harmonic, d spin down, n neutron, p proton. This input will 
yield a single antisymmetric matrix. This is done by using the signatures of the slater determinant constructed by the
specified orbitals. In this example our slater determinant would be:
~~~
|                                   |
| R0(r1)Y00(r1)dn   R0(r2)Y00(r2)dn |
|                                   |
| R0(r1)Y00(r1)dp   R0(r2)Y00(r2)dp |
|                                   |
~~~

The determinant of this matrix is:
~~~
+ R0(r1)Y00(r1)dn * R0(r2)Y00(r2)dp - R0(r1)Y00(r1)dp * R0(r2)Y00(r2)dn 
~~~

The +'s and -'s in front are the signatures and they are calculated based on the permutations of each of the single
particle orbitals. The signatures are gathered into an array:
~~~
signature array: [+1, -1]
~~~
The signature array is constructed once at the beginning of the calculation.

The spatial functions R and Y depend on the single particle coordinates which are evaluated during run time
where they are used to construct the orbital array:
~~~
orbital array: [R0(r1)Y00(r1)R0(r2)Y00(r2), R0(r1)Y00(r1)R0(r2)Y00(r2)]
~~~

Here you can see that both terms in the orbital array are equal and if you dot them 
with the signature array the wave function will vanish. But their spin and isospin configurations prevent this
by assigning them to different elements in the wave function matrix 
<img src="https://render.githubusercontent.com/render/math?math=A_{imn}">
(in this case i = 0 there is only one slater determinant). As a reminder the row and columns of A are determined by
the isospin and spin configurations respectively. So in addition to the orbital array and the signature array, the
indices for each of the elements of the orbital array in the wave function matrix are constructed using the
spin and isospin combinations from the slater determinant. In this example the spin and isospin part of the
determinant gives two terms:
~~~
spin isospin coordinates arrays:
determinant: -dndp + dpdn  is converted to:
spin index array: [dd, dd] --binary--> [00, 00] --base10--> [0, 0]
isospin index array: [np, pn] --binary--> [01, 10] --base10--> [1, 3] --rank order (due to charge conservation)-->  [0, 1]
~~~

so far we have the following data structures:
~~~
signature array: [+1, -1]
orbital array: [, R0(r1)Y00(r1)R0(r2)Y00(r2)]
spin index array: [0, 0]
isospin index array: [0, 1]
~~~

Next the wave function matrix is initialized with all zeros:

~~~
Starting wfc matrix 
A = 
     dd   du   ud   uu
   -------------------
np | 0    0    0    0 
pn | 0    0    0    0 
~~~

And the orbital array mutiplied by the signature array is added to the element of the wave function according to the
spin isospin indices: 

~~~
Amn construction pseudo code:

sig_orbs = signature_array * orbital_array
for i, j, sig_orb in zip(iso_indices, spin_indices, sig_orbs):
    A[i, j] += sig_orb
~~~

Resulting in
~~~
Starting wfc matrix 
A =
     dd                             du   ud   uu
   ----------------------------------------------
np | +R0(r1)Y00(r1)R0(r2)Y00(r2)    0    0    0 
pn | -R0(r1)Y00(r1)R0(r2)Y00(r2)    0    0    0 

~~~
If we were to example put both particles in the n isospin state, then the two terms in the determinant would
have had the same position in the matrix and their sum would result in zero as expected.

## Function Factory Pattern For Wave Function Construction
Wave functions in the code have the following function definitions:
~~~
def wave_function(parameters: jnp.array, particle_coordinates: jnp.array) -> matrix
~~~

Often times the behavior of different components of the wave function must be altered at run time based on 
the user input file. A simple example would be a function which loops over particle pairs will have more pairs
in 4He compared to 2H. To accomplish this we construct wave functions using a function factory pattern
(like python decorators) which return functions:
~~~python
import jax.numpy as jnp

def build_wave_function(particle_pair_indices):
    def wave_function(_params, r_coords):
        delta_r = r_coords[particle_pair_indices[:, 0]] - r_coords[particle_pair_indices[:, 1]] 
        delta_r = jnp.linalg.norm(delta_r, axis=0)
        return jnp.exp(- delta_r.sum() * _params[0])
    params = jnp.array([0])
    return wave_function, params
~~~
The builder function takes as input the particle pair indices for example: 
~~~
[[0, 1]
,[0, 2]
,[1, 2]]
~~~
and constructs a function that takes as input parameters and particle coordinates and calculates the
pair differences, their norm, their sum and then multiplies them by the first element in the params array.
The build function then returns the constructed wave function and the initial parameters for
that wave function. This pattern is used repeatedly to construct
the wave function. Combining two wave functions can be accomplished with the same pattern:

~~~python
import jax.numpy as jnp

def build_compound_wave_function(wfc1_args, wfc2_args):
    wfc1, wfc1_params = build_wfc1(*wfc1_args)
    n_wfc1_params = len(wfc1_params)
    wfc2, wfc2_params = build_wfc2(*wfc2_args)

    def compound_wave_function(_params, r_coords):
        w1 = wfc1(_params[:n_wfc1_params], r_coords)
        w2 = wfc2(_params[n_wfc1_params:], r_coords)
        return w1 * w2

    params = jnp.concatenate(wfc1_params, wfc2_params)
    return compound_wave_function, params
~~~

From the outside, this wave function behaves as if it only has one set of parameters (params) and it manages
slicing the parameters and feeding them to the correct wfcs. All of the details are managed inside of the
returned function. This pattern allows arbitrarily complex functions to be built in a modular way.
